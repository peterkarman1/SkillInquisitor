# Epic 2 Regression Harness Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a schema-first regression harness with self-contained fixtures, exact normalized finding comparison, safe baseline coverage, and repository guidance requiring relevant tests for meaningful changes.

**Architecture:** The implementation keeps the real scanner pipeline as the only execution path and builds the harness around it. Fixture-local `expected.yaml` files define behavior, `tests/fixtures/manifest.yaml` indexes and categorizes fixtures, and pytest helpers in `tests/conftest.py` handle discovery, validation, execution, normalization, and comparison. Documentation and repository policy land in the same epic so the harness becomes the default workflow, not optional tooling.

**Tech Stack:** Python 3.13, `pytest`, `pytest-asyncio`, `PyYAML`, existing `skillinquisitor` models/config/input/pipeline modules

Use `skillinquisitor.models` to synthesize findings for the second and third tests. Example:

```python
from skillinquisitor.models import Category, DetectionLayer, Finding, Location, Severity


def test_safe_fixture_matches_expected_contract(run_fixture_scan, assert_scan_matches_expected):
    result = run_fixture_scan("safe/simple-formatter")
    assert_scan_matches_expected("safe/simple-formatter", result)


def test_scoped_matching_ignores_out_of_scope_findings(
    build_expectation,
    assert_scan_matches_expected,
    empty_scan_result,
):
    expectation = build_expectation(
        verdict="SAFE",
        scope={"layers": ["deterministic"], "checks": ["D-1"]},
        findings=[],
    )
    result = empty_scan_result.model_copy(
        update={
            "findings": [
                Finding(
                    rule_id="ML-1",
                    layer=DetectionLayer.ML_ENSEMBLE,
                    category=Category.PROMPT_INJECTION,
                    severity=Severity.LOW,
                    message="out of scope",
                    location=Location(file_path="SKILL.md", start_line=1, end_line=1),
                )
            ]
        }
    )
    assert_scan_matches_expected(expectation, result)
```

## Execution Notes

- Keep `tests/conftest.py` focused on pytest-facing harness infrastructure. Do not turn it into a general utility module for unrelated test helpers.
- Do not add fake malicious fixtures for checks that do not exist yet.
- Prefer small, readable fixture content over clever synthetic complexity.
- If exact comparison becomes noisy, tighten the normalized projection rather than weakening the contract.
- If a documentation update reveals a divergence from the requirements docs, update those docs in the same change rather than leaving the mismatch behind.
